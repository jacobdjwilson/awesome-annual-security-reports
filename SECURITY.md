# Security Policy

## Reporting a Vulnerability

We take the security of the documents and links hosted in this repository seriously. While we utilize automated GitHub Actions to upload reports to VirusTotal for confidence, we acknowledge that false negatives can occur.

### What to Report
Please report the following issues immediately:
* **Malicious PDFs:** Any report in the repository that triggers antivirus software or contains exploit code.
* **Phishing/Malicious Links:** Any URL in the `README.md` that redirects to malicious content or phishing sites.
* **Copyright Infringement:** If a report requiring a paid subscription has been inadvertently uploaded (violating our Disclaimer).

### How to Report
1.  **Open an Issue:** Please open a new issue in the repository.
2.  **Tagging:** Label the issue as `security` or `malware` if possible.
3.  **Details:** Include the filename of the report or the specific link in question. If reporting a malicious file, please provide a link to a VirusTotal analysis if you have already performed one.

### Response Timeline
* You can expect an acknowledgement of your report within **48 hours**.
* If a file is confirmed to be malicious, it will be removed from the repository history immediately.
* If a link is confirmed to be malicious, the `README.md` will be patched immediately.

**Disclaimer:** This list is a community resource. While we strive to verify all content, users should always exercise caution when downloading files or clicking external links.
